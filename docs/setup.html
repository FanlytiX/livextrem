Setup

Vollständige Einrichtung der Projektumgebung für Datenbank, Twitch-Anbindung, Quellcode, Paketinstallation und lokalen Start von liveXtrem.

Info: Diese Seite beschreibt nicht nur die Nutzung der fertigen EXE, sondern die komplette technische Einrichtung für die Projektumgebung. Sie führt nacheinander durch Vorbereitung, Datenbank, Twitch-API, Projekt-Download, Paketinstallation, Konfiguration und Start.
Voraussetzungen Welche Programme und Rahmenbedingungen vor dem Start vorhanden sein müssen. Datenbank mit XAMPP MariaDB lokal starten, Datenbank anlegen, SQL importieren und Benutzerrechte setzen. Twitch Developer Console Twitch-Anwendung registrieren, Redirect-URI setzen und API-Zugangsdaten übernehmen. Projekt herunterladen UV installieren, Repository klonen und in das richtige Verzeichnis wechseln. Konfiguration Die Datei keydata.env korrekt befüllen und Datenbank- und Twitch-Werte sauber eintragen. Projektstart liveXtrem mit login.py starten und die erste Registrierung durchführen.

Ziel dieser Seite

Diese Setup-Seite richtet die vollständige lokale Projektumgebung für liveXtrem ein. Dazu gehören die lokale Datenbank über XAMPP, die Anbindung an Twitch über eine eigene Developer-Anwendung, das Herunterladen des Repositories, die Installation der Python-Abhängigkeiten mit UV, das Eintragen der Konfigurationswerte in der Env-Datei und der anschließende Start der Anwendung. Die Reihenfolge ist so aufgebaut, dass auch eine technisch interessierte Person ohne tiefes IT-Vorwissen die Einrichtung Schritt für Schritt abarbeiten kann.

Voraussetzungen

Für die vollständige Einrichtung werden einige Werkzeuge benötigt. XAMPP ist für liveXtrem der wichtigste Sonderfall, weil darüber die lokale MariaDB und phpMyAdmin bereitgestellt werden. Git, UV, VS Code und die Python-Erweiterung werden für den Quellcode-Download, die Paketinstallation und die Bearbeitung der Konfigurationsdatei benötigt.

Datenbank mit XAMPP einrichten

liveXtrem speichert Benutzerkonten, Rollen, Stream-Planung, Finanzdaten und weitere Projektdaten in einer relationalen Datenbank. Für die lokale Umgebung übernimmt XAMPP diese Aufgabe, weil darüber eine MariaDB-Instanz und phpMyAdmin ohne zusätzlichen Serveraufbau bereitgestellt werden. XAMPP bringt die benötigten Werkzeuge gebündelt mit, sodass die Datenbank für dieses Projekt lokal auf dem eigenen Rechner laufen kann.

1. XAMPP herunterladen und installieren

  1. Öffnen Sie die offizielle XAMPP-Seite unter https://www.apachefriends.org/de/index.html.
  2. Laden Sie die Windows-Version von XAMPP herunter.
  3. Starten Sie die Installationsdatei und folgen Sie dem Installationsassistenten. Die Standardoptionen sind für dieses Projekt in der Regel ausreichend.
  4. Öffnen Sie nach Abschluss die Anwendung XAMPP Control Panel.
Info: In XAMPP läuft die Datenbank technisch als MariaDB. Im Control Panel und in älteren Bezeichnungen taucht oft trotzdem noch MySQL auf. Für dieses Setup ist damit der Datenbankdienst gemeint, den liveXtrem verwendet.

2. Benötigte Module starten

  1. Starten Sie im XAMPP Control Panel die Module Apache und MySQL.
  2. Wenn beide laufen, sind die Zeilen im Control Panel als aktiv markiert.
  3. Öffnen Sie zur Kontrolle im Browser http://localhost/. Wenn die XAMPP-Startseite erscheint, läuft Apache korrekt.
  4. Öffnen Sie anschließend http://localhost/phpmyadmin/. Damit gelangen Sie in die Datenbankverwaltung.
Wichtig: Solange liveXtrem auf die Datenbank zugreift, muss der Datenbankdienst in XAMPP laufen. Praktisch heißt das: Vor dem Start von liveXtrem XAMPP öffnen und prüfen, dass MySQL aktiv ist. Für Arbeiten mit phpMyAdmin muss zusätzlich Apache laufen.

3. Leere Datenbank livextrem anlegen

  1. Öffnen Sie in phpMyAdmin die Startseite.
  2. Wählen Sie den Bereich Neu beziehungsweise New, um eine neue Datenbank anzulegen.
  3. Vergeben Sie als Namen exakt livextrem.
  4. Legen Sie die Datenbank an. Wenn eine Zeichencodierung abgefragt wird, ist utf8mb4 passend.

4. SQL-Datei importieren

Die passende SQL-Datei liegt im Projekt unter database/livextrem.sql. Wenn das Repository bereits geklont wurde, kann genau diese Datei direkt verwendet werden. Falls das Projekt noch nicht lokal vorliegt, kann die gleiche Datei auch aus dem GitHub-Repository heruntergeladen und anschließend lokal importiert werden.

  1. Klicken Sie in phpMyAdmin links auf die zuvor erstellte Datenbank livextrem.
  2. Wechseln Sie oben zum Reiter Importieren beziehungsweise Import.
  3. Wählen Sie die Datei database/livextrem.sql aus.
  4. Starten Sie den Import mit der Standardschaltfläche am Seitenende.
  5. Warten Sie, bis phpMyAdmin den erfolgreichen Import bestätigt.
Tipp: Nach dem Import sollten Tabellen wie users, roles, streamer, user_roles und twitch_tokens in der Datenbank sichtbar sein. Wenn diese Tabellen fehlen, wurde die SQL-Datei nicht vollständig importiert.

5. Datenbankbenutzer anlegen und berechtigen

  1. Öffnen Sie in phpMyAdmin den Bereich Benutzerkonten beziehungsweise User accounts.
  2. Legen Sie einen neuen Benutzer an.
  3. Verwenden Sie als voreingestellte Projektdaten livextrem als Benutzername und livextrem1243 als Passwort.
  4. Setzen Sie den Host des Benutzers so, dass er zur lokalen Verbindung passt. Für die Standardkonfiguration des Projekts ist 127.0.0.1 passend.
  5. Vergeben Sie diesem Benutzer alle Rechte auf die Datenbank livextrem.
  6. Speichern Sie den Benutzer.
Voreinstellung Wert Bedeutung
Datenbankname livextrem Name der lokal angelegten Datenbank in phpMyAdmin.
Datenbankbenutzer livextrem Lokaler Datenbanknutzer, mit dem sich liveXtrem verbindet.
Datenbankpasswort livextrem1243 Passwort des lokalen Datenbanknutzers.
Host 127.0.0.1 Lokale Rechneradresse für die Verbindung zur XAMPP-Datenbank.
Port 3306 Standardport für MariaDB/MySQL in einer lokalen XAMPP-Installation.

Diese Werte können geändert werden. In dem Fall müssen die tatsächlichen Datenbankwerte und die spätere Projektkonfiguration in livextrem/keydata.env aber exakt zusammenpassen. Abweichende Benutzer, Passwörter, Ports oder Hosts sind erlaubt, solange sie an beiden Stellen gleich eingetragen werden.

Twitch Developer Console und API einrichten

liveXtrem nutzt Twitch OAuth, um ein Twitch-Konto mit dem lokalen Benutzer zu verknüpfen. Dafür wird eine eigene Anwendung in der Twitch Developer Console registriert. Aus dieser Registrierung stammen die beiden zentralen Werte Client ID und Client Secret, die anschließend in die Projektkonfiguration übernommen werden.

1. Twitch-Konto vorbereiten

  1. Öffnen Sie die Entwicklerkonsole unter https://dev.twitch.tv/console.
  2. Melden Sie sich mit dem Twitch-Konto an, das für die Registrierung der Anwendung verwendet werden soll.
  3. Stellen Sie sicher, dass das Twitch-Konto verifiziert ist und die Zwei-Faktor-Authentifizierung aktiviert wurde. Ohne diese Freischaltung lässt Twitch keine App-Registrierung zu.

2. Twitch-Anwendung registrieren

  1. Wechseln Sie in der Developer Console zum Bereich Applications.
  2. Klicken Sie auf Register Your Application.
  3. Vergeben Sie einen eindeutigen Namen für die Anwendung.
  4. Tragen Sie bei OAuth Redirect URLs exakt http://localhost:8080 ein und füge die Adresse hinzu.
  5. Wählen Sie eine passende Kategorie für eine lokale Projekt- oder Desktop-Anwendung.
  6. Schließen Sie die Registrierung ab.
Wichtig: Die Redirect-URI muss exakt mit dem Projektwert übereinstimmen. Für liveXtrem ist standardmäßig http://localhost:8080 hinterlegt. Eine andere Schreibweise wie http://127.0.0.1:8080, ein anderer Port oder ein zusätzlicher Slash am Ende führen dazu, dass Twitch den OAuth-Callback ablehnt.

3. Client ID und Client Secret übernehmen

  1. Öffnen Sie nach dem Erstellen im Bereich Developer Applications die soeben angelegte Anwendung.
  2. Gehe auf Manage.
  3. Kopieren Sie die Client ID.
  4. Erzeugen Sie über New Secret ein Client Secret und kopiere es sofort an einen sicheren Ort.
Twitch-Wert Eintrag in livextrem/keydata.env Zweck im Projekt
Client ID LIVEXTREM_TWITCH_CLIENT_ID Identifiziert die registrierte Twitch-Anwendung gegenüber der API.
Client Secret LIVEXTREM_TWITCH_CLIENT_SECRET Wird beim OAuth-Austausch für den sicheren Tokenabruf benötigt.
Redirect URI LIVEXTREM_TWITCH_REDIRECT_URI Muss mit der in Twitch registrierten Callback-Adresse übereinstimmen. Standardwert: http://localhost:8080.

Die Datei livextrem/config.py lädt diese Einträge automatisch aus livextrem/keydata.env. Die Werte werden also nicht direkt in Python-Dateien eingetragen, sondern ausschließlich in der Env-Datei gepflegt.

Projekt herunterladen

Bevor die Abhängigkeiten installiert oder die Konfiguration bearbeitet werden, muss das Repository lokal vorliegen. Der erste Schritt dafür ist die Installation von UV, weil UV anschließend sowohl die Abhängigkeiten installiert als auch das Projekt in der passenden Umgebung ausführen kann.

1. UV installieren

  1. Öffnen Sie unter Windows eine PowerShell oder das integrierte Terminal in VS Code.
  2. Führen Sie den offiziellen Installationsbefehl aus:
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Der Befehl lädt den offiziellen Windows-Installer für uv herunter und führt ihn direkt aus. Anschließend kann mit einem kurzen Versionscheck geprüft werden, ob die Installation verfügbar ist.

uv --version
Tipp: Wenn der Befehl uv --version direkt danach noch nicht gefunden wird, schließen Sie das Terminal vollständig und öffnen Sie es neu. Unter Windows wird der neue Pfad oft erst nach einem neuen Terminalstart erkannt.

2. Repository klonen

  1. Öffnen Sie ein Terminal in dem Ordner, in dem das Projekt gespeichert werden soll.
  2. Führen Sie dann die beiden Befehle nacheinander aus:
git clone https://github.com/FanlytiX/livextrem
cd livextrem

git clone lädt das gesamte Repository lokal herunter. Dazu gehören der Python-Code, die SQL-Datei, die Env-Datei, die Dokumentation sowie die Projektdefinition mit pyproject.toml und uv.lock. Mit cd livextrem wechseln Sie in das Verzeichnis, in dem die nächsten Befehle ausgeführt werden müssen.

Hinweis: Öffnen Sie genau diesen geklonten Ordner anschließend in VS Code. So finden Sie die spätere Konfigurationsdatei livextrem/keydata.env direkt im Projektbaum.

Pakete mit UV installieren

Im Repository ist bereits festgelegt, welche Python-Abhängigkeiten liveXtrem benötigt. Deshalb werden die Pakete nicht einzeln mit mehreren pip install-Befehlen nachgezogen, sondern gesammelt mit uv sync installiert.

  1. Stellen Sie sicher, dass Sie sich im geklonten Projektordner befinden, also dort, wo auch pyproject.toml und uv.lock liegen.
  2. Führen Sie anschließend aus:
uv sync

Dieser Schritt richtet die Projektumgebung ein und installiert die im Repository definierten Abhängigkeiten. Dazu gehören unter anderem customtkinter, Pillow, mariadb, mysql-connector-python, requests, tkcalendar, reportlab, twitchapi, mkdocs und python-dotenv.

Wichtig: uv sync muss im Projekt-Wurzelverzeichnis ausgeführt werden. Wenn der Befehl in einem anderen Ordner gestartet wird, fehlen die Projektdefinition und das Lockfile, und die Umgebung wird nicht korrekt aufgebaut.

Projekt konfigurieren

Die Konfiguration von liveXtrem liegt in der Datei livextrem/keydata.env. Dort stehen die Datenbankwerte und die Twitch-Zugangsdaten. Genau diese Datei ist die zentrale Stelle für installationsabhängige Werte. Der Code liest die Einträge über livextrem/config.py ein und verwendet sie anschließend beim Datenbankzugriff und beim Twitch-OAuth.

Konfigurationsdatei öffnen

  1. Öffnen Sie das geklonte Repository in VS Code.
  2. Wechseln Sie im Explorer in den Unterordner livextrem.
  3. Öffnen Sie die Datei keydata.env.
  4. Tragen Sie dort Ihre Datenbank- und Twitch-Werte ein beziehungsweise prüfen Sie die vorhandenen Standardwerte.

Welche Einträge verwendet werden

Eintrag Standardwert Wofür der Wert steht
LIVEXTREM_DB_HOST 127.0.0.1 Adresse des lokalen Datenbankservers. Für XAMPP auf demselben Rechner ist 127.0.0.1 passend.
LIVEXTREM_DB_PORT 3306 Port der lokalen MariaDB/MySQL-Instanz.
LIVEXTREM_DB_NAME livextrem Name der Datenbank, die zuvor in phpMyAdmin angelegt wurde.
LIVEXTREM_DB_USER livextrem Benutzername für die Datenbankverbindung.
LIVEXTREM_DB_PASS livextrem1243 Passwort des Datenbankbenutzers.
LIVEXTREM_TWITCH_CLIENT_ID leer Hier wird die Client ID aus der Twitch Developer Console eingetragen.
LIVEXTREM_TWITCH_CLIENT_SECRET leer Hier wird das Client Secret aus der Twitch Developer Console eingetragen.
LIVEXTREM_TWITCH_REDIRECT_URI http://localhost:8080 Redirect-URI für den OAuth-Callback. Dieser Wert muss mit der Twitch-App identisch sein.
Bereits vorbelegt: Die Standardwerte für Host, Port, Datenbankname, Datenbankbenutzer, Datenbankpasswort und Redirect-URI sind in der mitgelieferten keydata.env bereits eingetragen. Ausfüllen müssen Sie mindestens LIVEXTREM_TWITCH_CLIENT_ID und LIVEXTREM_TWITCH_CLIENT_SECRET. Wenn Sie Ihre Datenbankdaten abweichend gewählt haben, tragen Sie auch diese Werte hier passend um.
LIVEXTREM_DB_HOST="127.0.0.1"
LIVEXTREM_DB_PORT="3306"
LIVEXTREM_DB_NAME="livextrem"
LIVEXTREM_DB_USER="livextrem"
LIVEXTREM_DB_PASS="livextrem1243"
LIVEXTREM_TWITCH_CLIENT_ID="IHRE_CLIENT_ID"
LIVEXTREM_TWITCH_CLIENT_SECRET="IHR_CLIENT_SECRET"
LIVEXTREM_TWITCH_REDIRECT_URI="http://localhost:8080"

Wenn Sie Host, Port, Datenbankname, Benutzername, Passwort oder Redirect-URI ändern, müssen die Werte exakt zu Ihrer realen Umgebung passen. Das Projekt prüft diese Einträge beim Start. Leere oder widersprüchliche Werte führen dazu, dass die Datenbankverbindung oder der Twitch-Login fehlschlagen.

Projekt starten

Der Einstiegspunkt der Anwendung ist die Datei login.py im Unterordner livextrem. Gestartet wird das Projekt am einfachsten aus dem Repository-Wurzelverzeichnis heraus, damit UV die bereits eingerichtete Umgebung direkt verwenden kann.

  1. Öffnen Sie ein Terminal im geklonten Projekt-Wurzelverzeichnis.
  2. Prüfen Sie noch einmal, dass XAMPP läuft und der Datenbankdienst aktiv ist.
  3. Starten Sie dann die Anwendung mit folgendem Befehl:
uv run livextrem/login.py

Damit führt UV die Datei livextrem/login.py innerhalb der synchronisierten Projektumgebung aus. Genau dieser Startweg ist für die lokale Einrichtung am saubersten, weil dabei dieselben Abhängigkeiten genutzt werden, die zuvor mit uv sync installiert wurden.

Erster Start bei einer frischen Datenbank

Nach einem frischen SQL-Import ist die Tabellenstruktur vorhanden, aber es existiert noch kein normaler Benutzer für die Anmeldung. Deshalb wird beim ersten Start typischerweise zuerst die Registrierung verwendet.

  1. Klicken Sie im Startfenster auf Registrieren.
  2. Tragen Sie Benutzername, E-Mail-Adresse und Passwort ein.
  3. Klicken Sie auf Mit Twitch verbinden & registrieren.
  4. Der Browser öffnet die Twitch-Anmeldung. Melden Sie sich dort mit dem Twitch-Konto an, das mit dem Streamer-Profil verknüpft werden soll.
  5. Nach erfolgreicher Rückmeldung erstellt liveXtrem den lokalen Benutzer und speichert die Twitch-Zuordnung in der Datenbank.
  6. Wechseln Sie danach zurück zum Login-Fenster und melden Sie sich mit den gerade angelegten Zugangsdaten an.
Wichtig: Die Registrierung funktioniert nur dann sauber, wenn die Datenbank erreichbar ist, die SQL-Datei importiert wurde und in keydata.env bereits LIVEXTREM_TWITCH_CLIENT_ID und LIVEXTREM_TWITCH_CLIENT_SECRET eingetragen sind.

Fehlerquellen, Hinweise und wichtige Tipps

!

XAMPP oder MySQL läuft nicht

Wenn der Datenbankdienst nicht aktiv ist, kann liveXtrem keine Verbindung aufbauen. Vor jedem Start prüfen, ob im XAMPP Control Panel MySQL aktiv ist.

!

Falsche Datenbankdaten

Host, Port, Datenbankname, Benutzer und Passwort in keydata.env müssen exakt zu der lokal angelegten Datenbank passen.

!

Twitch-Werte fehlen

Leere oder falsche Einträge bei LIVEXTREM_TWITCH_CLIENT_ID und LIVEXTREM_TWITCH_CLIENT_SECRET verhindern die Registrierung und den Twitch-Login.

!

Redirect-URI stimmt nicht

Die in Twitch registrierte Redirect-URI und LIVEXTREM_TWITCH_REDIRECT_URI müssen absolut identisch sein. Schon kleine Abweichungen blockieren OAuth.

!

Falscher Projektordner

uv sync und uv run livextrem/login.py gehören in den Repository-Wurzelordner. Wer im falschen Verzeichnis startet, arbeitet an der Projektstruktur vorbei.

!

SQL-Datei nicht importiert

Wenn die Tabellen fehlen, kann die Anwendung weder Benutzer anlegen noch Rollen, Streamer oder Twitch-Daten speichern. In phpMyAdmin immer kontrollieren, ob die Tabellen wirklich vorhanden sind.

!

Benutzer ohne Rechte

Der Datenbankbenutzer braucht vollständige Rechte auf livextrem. Ohne ausreichende Rechte scheitern bereits einfache Lese- und Schreibzugriffe.

!

Terminal nicht neu gestartet

Nach der UV- oder Git-Installation erkennt Windows neue Befehle manchmal erst in einem neu geöffneten Terminalfenster.
